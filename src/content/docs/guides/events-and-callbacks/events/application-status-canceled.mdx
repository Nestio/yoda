The "Application Canceled" event is a significant webhook triggered when an application is canceled within Funnel Leasing. This event is essential for tracking and managing application statuses, providing necessary data for integrations to update records and take appropriate actions.

## Event Details

- **Event Type**: `application_canceled`
- **Description**: This event is triggered when an application is canceled, indicating that the application process has been terminated.
- **Event Payload Example**:

```json
{
  "event_id": "evt_1PabcKLt4dXK03v5qaLmnopQ",
  "event_type": "application_canceled",
  "created_at": 1682345678,
  "event_data": {
      "application_id": 4444,
      "renter_id": 4444,
      "canceled_by": "leasing_agent_002",
      "community_id": 65432
  }
}
```

## Payload Fields

- **`event_id`**: A unique identifier for the event.
- **`event_type`**: The type of event (`application_canceled`).
- **`created_at`**: A timestamp indicating when the event occurred.
- **`event_data`**: Contains additional data related to the event:
  - **`application_id`**: The ID of the canceled application.
  - **`renter_id`**: The proprietary renter ID within Funnel.
  - **`canceled_by`**: The identifier of the leasing agent or system that canceled the application.
  - **`community_id`**: The ID of the community where the application was canceled.

## Use Cases

This event is vital for integrations that need to track and manage application cancellations. For instance, a Property Management System (PMS) integration can utilize this event to:

- **Update Application Status**: Automatically update the status of the application to 'canceled' within the system.
- **Release Reserved Units**: Update the availability status of any units reserved by the canceled application to make them available for other applicants.
- **Data Synchronization**: Sync the cancellation data across multiple systems to maintain consistent and up-to-date records.
- **Notify Stakeholders**: Send notifications to relevant stakeholders, including leasing agents and property managers, about the cancellation.

By leveraging this event, integrations can streamline the cancellation process, ensure accurate unit availability, and maintain data integrity within the Funnel Leasing ecosystem.
