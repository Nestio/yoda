The "Application Reserved Unit" event is a pivotal webhook triggered when an application successfully reserves a unit within Funnel Leasing. This event is essential for tracking and managing the reservation process, providing necessary data for integrations to act in real-time.

## Event Details

- **Event Type**: `application_unit_reserved`
- **Description**: This event is triggered when an application successfully reserves a unit, signaling the reservation's confirmation.
- **Event Payload Example**:

```json
{
  "event_id": "evt_1MqqbKLt4dXK03v5qaIbiNCC",
  "event_type": "application_unit_reserved",
  "created_at": 1680064028,
  "event_data": {
      "application_id": 1111,
      "client_id": 1111,
      "community_id": 43522
  }
}
```

## Payload Fields

- **`event_id`**: A unique identifier for the event.
- **`event_type`**: The type of event (`application_unit_reserved`).
- **`created_at`**: A timestamp indicating when the event occurred.
- **`event_data`**: Contains additional data related to the event:
  - **`application_id`**: The ID of the application that reserved the unit.
  - **`client_id`**: The proprietary client ID within Funnel.
  - **`community_id`**: The ID of the community where the reserved unit is located.

## Use Cases

This event is vital for integrations that require tracking and managing unit reservations. For instance, a Property Management System (PMS) integration can utilize this event to:

- **Update Availability**: Automatically update the availability status of units to prevent double bookings and ensure accurate inventory.
- **Reservation Confirmation**: Send confirmation notifications to applicants and relevant stakeholders.
- **Data Synchronization**: Sync reservation data across multiple systems to maintain consistent and up-to-date records.

By leveraging this event, integrations can enhance operational efficiency, improve user experience, and maintain data integrity within the Funnel Leasing ecosystem.
