<div class="grid grid-cols-2 gap-4">
  <div class="col-span-1 justify-self-start">
    <p>
      Funnel Leasing uses conventional HTTP response codes to indicate the success or failure of an API request. Here is a general overview:
    </p>
    <ul>
      <li>Codes in the <code>2xx</code> range indicate success.</li>
      <li>Codes in the <code>4xx</code> range indicate an error due to the provided information (e.g., a required parameter was omitted, a permission error, etc.).</li>
      <li>Codes in the <code>5xx</code> range indicate a server error on Funnel Leasing’s end (these are rare).</li>
    </ul>
    <p>
      Some <code>4xx</code> errors (e.g., a missing parameter) include an error code that briefly explains the issue along with a human-friendly error message.
    </p>

    #### Attributes

---

<ul>
  <li><strong>code</strong>: <span style="font-size: smaller;"><code>integer</code></span></li>
  <p>
    This represents the specific error code returned by the API. Each code corresponds to a specific type of error, helping to identify and troubleshoot issues quickly.
  </p>

{" "}
<li>
  <strong>message</strong>:{" "}
  <span style="font-size: smaller;">
    <code>nullable string</code>
  </span>
</li>
<p>
  A human-readable message that provides more details about the error. This
  message can help users understand the nature of the error and potential steps
  to resolve it.
</p>

  <li><strong>param</strong>: <span style="font-size: smaller;"><code>nullable string</code></span></li>
  <p>
    If the error is related to a specific parameter in the API request, this field will indicate which parameter caused the error. This is particularly useful for identifying issues with request formatting or missing information.
  </p>
</ul>
